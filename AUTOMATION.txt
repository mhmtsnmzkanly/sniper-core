# 1. Stratejik Hedef Tanımı

## Ürün Seviyesi Hedef

* Cross-platform (Win / macOS / Linux)
* Headful + Headless destek
* Deterministic execution (aynı DSL → aynı davranış)
* DSL versioning & backward compatibility
* Debuggable + observable runtime

```
   ↓

```

# 3. Core Refactor Önerisi

## 3.1 Engine’i 3'e Böl

Şu anda executor mantığı tek yerde yoğunlaşıyor. Bunu net ayır:

### A. Planner (Stateless)

Görevi:

* DSL’i normalize etmek
* Variable interpolation compile-time çözmek
* Syntax validation yapmak
* Execution Plan üretmek

Çıktı:

```rust
ExecutionPlan {
    steps: Vec<ExecutableStep>
}
```

---

### B. Runtime (Stateful)

Görevi:

* AutomationContext’i yönetmek
* Async execution
* Error boundaries
* Cancellation handling
* Retry logic

---

### C. Driver Abstraction

CDP şu an tightly coupled görünüyor.

Şöyle bir trait ekle:

```rust
#[async_trait]
pub trait AutomationDriver {
    async fn navigate(&self, url: &str) -> Result<()>;
    async fn click(&self, selector: &str) -> Result<()>;
    async fn type_text(&self, selector: &str, value: &str) -> Result<()>;
    async fn eval(&self, js: &str) -> Result<String>;
    async fn screenshot(&self) -> Result<Vec<u8>>;
}
```

Bugün: CDPDriver
Yarın: WebDriverDriver
Sonra: Native OS automation driver

Bu soyutlama ileride kritik olacak.

# 4. DSL Olgunlaştırma Planı

## 4.1 DSL Versioning

Şu an enum DSL var. Bunu future-proof yap:

```rust
#[derive(Serialize, Deserialize)]
pub struct AutomationScript {
    pub version: u32,
    pub metadata: ScriptMetadata,
    pub steps: Vec<Step>,
}
```
---

## 4.2 Scoped Variables (Önemli)

Şu an global scope var. Bu ileride kaos yaratır.

Yeni model:

```
Global Scope
 └── Loop Scope
      └── Conditional Scope
```

Implementation:

```rust
struct ScopeStack {
    stack: Vec<HashMap<String, String>>
}
```

Lookup:

* Top-down resolve
* En üst scope override eder

---

# 5. Execution Model Geliştirme

## 5.1 Deterministic Async Model

Şu an tokio task spawn ediliyor. Güzel.

Ama engine içinde:

* Parallel step execution YASAK
* Strict sequential await
* Timeout per step
* CancellationToken destekli execution

Öneri:

```rust
struct ExecutionConfig {
    step_timeout: Duration,
    retry_policy: RetryPolicy,
    screenshot_on_error: bool,
}
```

---

## 5.2 Retry Engine

Her step için:

```rust
RetryPolicy {
    max_attempts: 3,
    backoff: Exponential,
}
```

Bu özellikle:

* WaitSelector
* Click
* Network idle

için hayati.

---

# 6. Hata Modeli

Şu an hata = abort.

Yeni model:

```rust
enum StepErrorStrategy {
    Abort,
    Continue,
    Retry,
    Fallback(Vec<Step>),
}
```

DSL içine opsiyonel alan olarak eklenebilir:

```rust
Click {
   selector: String,
   on_error: Option<StepErrorStrategy>
}
```

# 9. Frame & DOM Stabilite

Şu an frame desteği zayıf.

Öneri:

* Recursive querySelectorAll wrapper
* Frame-aware selector syntax:

```
iframe#login >> button.submit
```

Parser:

* ">>" ile frame boundary parse edilir
* Engine recursive resolve yapar

# 10. Gelişmiş DSL Özellikleri (Enterprise Seviye)

## 10.1 Script Functions

Reusable block:

```yaml

  login:
    - type: Type
    - type: Click

steps:
  - call: login
```

---

## 10.2 Data Pipeline Mode

Dataset-first automation:

```
For each row:
   inject variables
   run steps
   append result
```

Bu scraping senaryoları için çok güçlü olur.

# 11. Güvenlik

CDP injection yapıyorsun. Riskler:

* XSS style script injection
* Malicious DSL payload

Öneri:

* JS wrapper sandbox
* DSL signature validation
* Max script length sınırı
